Clinical Voice Assistant Prototype

A planned feature of my clinical software product was an intelligent voice assistant.

We researched the approaches of Amazon's Alexa, and the Google Assistant

We used the concept of user intents, slots, etc. to document what the voice assistant would support. This also helped me start to determine the best ways to communicate UX learnings to development in a format that would be most beneficial for them.

As our voice assistant research proceeded, I realized there it could be beneficial to expand on and share the UX+AI knowledge I was acquiring. I created a talk for World Usability Day inside of our company talking about how UX should think about and approach design for artificial intelligence, because it is a new area with non-traditional design needs. You can view my slides for that talk here.

A working prototype that we could iterate on would be the most helpful to us in user research and testing. After some exploration of various tools, I figured out that I could use a real natural language engine combined with tools that I have built previously.

I settled on the Snips NLU open source Python Library (see https://snips-nlu.readthedocs.io/en/latest/) as the part that would parse the user's speech into a meaningful intent. An interesting characteristic of Snips NLU is that the training data sets are all in yaml files, which are both easy to manually edit and easy to parse. To expose the natural language processing through a Web API, I used the Django Python Web Framework, which allowed me to keep all of the back end in the same Python project.

The portion of this solution that users would interact with for testing was an Angular application that made calls to the Django Web application as appropriate.

Because the voice assistant feature for our product would be dealing with patient health information, I was able to use the patient data generator tool that I had previously created. This added a sense of realism to the data displayed by the voice assistant protottype.
