Carequality Research and UI Design

The CareQuality application was a new product for internal configuration and management of customers on the CareQuality network.

My role on this was the UI design of all parts of the CareQuality application; the user interface was to be written in Angular.

For requirements, project manager gave me a combination of Word documents and Balsamiq mock ups. Some requirements also sent via email. So, I often needed to deconstruct Balsamiq designs in order to understand requirements

For example in the screen shot above, I needed to first determine the user's task, which was to add a participant to a list of linked providers.

From the user's perspective this should involve just a simple text-based search for a provider (via an 'add' button in the screen shot below), and then just select to add them to the list. The screen shot below shows my design based on the wireframe and my reinterpretation of the requirement.

Some functionality changes and additions would come in during development and I would update the designs, calling out the work item number, as shown in the screen shot below.

The development team chose to use the UI Starter Kit that I created. As this was the first team that I worked closely with on the starter kit, I learned how a team would actually use the kit in a product. So, in parallel with the UI design work I was doing, I was also improving and streamlining the UI Starter Kit to better fit developers' usage of it.
