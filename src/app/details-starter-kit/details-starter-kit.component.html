UI Starter Kit

What is it?

  • A collection of UI related functionality, provided as NPM packages
  • Common styles, graphics, fonts, and UI components that can be used by developers across the company. I created Custom css/scss themes for the Kendo UI toolkit and for ngBootstrap so that developers can align with the UX guidelines simply by installing the theme.
  • Using the NPM model allows developers to easily update packages with no disruption to their application development.

Why is it needed?

  • Products across Allscripts use different technologies and architectures, and not all products can completely align with the UX style guidelines. Different development teams need different combinations of packages.
  • So, the starter kit is designed on a "cafeteria" model: use only the packages that are appropriate for your product.

Packages contained in the Starter Kit

Customized Kendo Theme Numerous development teams use Kendo UI for Angular (https://www.telerik.com/kendo-angular-ui), and so I created a custom Kendo UI theme that aligns with the UX Visual Guidelines.
Custom Angular Components A number of components in our UX guidelines are not available in either the ngBootstrap or Kendo component set. These custom components fill that gap.
UX Common Icon Set Our UX team has created a set of icons that are used across all of our products. I transformed all of the individual images into a web font (similar to how FontAwesome works), which is very easy for development to use and update as we add new icons.
Fonts Our team wanted to ensure a consistent typography across applications, and chose Roboto as the font we would recommend. Typically, a web application may simply link to the Google fonts site to get the needed web font files. However, many clients that use our products are not directly connected to the Internet. So I was able to obtain legal approval to distribute the font files with the web application to ensure the fonts would be available.
