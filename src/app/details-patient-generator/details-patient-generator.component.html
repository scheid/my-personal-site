Patient Record Generator

One of the challenges in healthcare IT User Interface design is knowing what the designs will look and behave like when populated with real data. The type and amount of data can greatly affect how usable and readable the designs are.

This is the motivation behind the patient data generator. I created this tool for designers creating static designs and coded prototypes. The data schema also complies with an internal format used by our EHR's so testing teams can also use the service to populate EHR test environments with realistic data. For static prototypes, an SVG file format of the patient record can be downloaded. For coded prototypes, a json file containing the patient data is available. This tool uses techniques similar to a Monte Carlo simulation to build up all of the parts of a patient record. The generator is architected to be infinitely extensible through a set of data and configuration files (csv and yaml files) Any number of new patient profiles can be created and existing ones can all be edited to tweak the generator results.

Details; how the generator works

The generation process starts with a profile, which is a yaml file, and the various parts of the record are generated based on the contents of that profile. For example, a diabetes profile may have general problems defined: diabetes mellitus, neuropathy, foot ulcer, elevated PSA, and upper respiratory infection. A medication can be linked to each of those problems as well.

Vital sign value ranges are defined in the profile, and a random value is generated within that range for each vital sign. Choosing random values in this way lends realism (because the values are a bit messy as in real life), and credibility (the values are believable for the patient profile). The number of sets of vitals and the date intervals can also be controlled (e.g., if you want to include a set of vitals every month for the past two years.

Patient demographic information is selected randomly from a set of about 15,000 non-existent patients.

All of the above factors combine to ensure that many patient records can be generated for the same profile, and they will all be a bit different.

Finally, regarding performance, a single patient record can typically be generated in less than half a second. This is of course system dependent, but the time would be approximately that.
